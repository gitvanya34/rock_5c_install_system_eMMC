# rock_5c_install_system_eMMC
Radxa Rock5c Install the system on the eMMC Module without reader

Установка операционной системы на eMMC модуль Radxa Rock 5c rk3588s2 без использования ридера

## Установка и использование rkdeveloptool

## Описание
**rkdeveloptool** — это инструмент от Rockchip для работы с устройствами через USB на платформах Linux/macOS. Его можно рассматривать как открытую версию утилиты **upgrade_tool**.

Если ваша операционная система не предоставляет **rkdeveloptool**, вам потребуется скомпилировать и установить его из исходного кода.

---
## Установка

### Debian / Ubuntu / ArchLinux
Выполните следующие команды для установки необходимых пакетов и компиляции:
```sh
sudo apt-get update
sudo apt-get install -y libudev-dev libusb-1.0-0-dev dh-autoreconf pkg-config libusb-1.0 build-essential git wget

git clone https://github.com/rockchip-linux/rkdeveloptool
cd rkdeveloptool

wget https://patch-diff.githubusercontent.com/raw/rockchip-linux/rkdeveloptool/pull/73.patch
wget https://patch-diff.githubusercontent.com/raw/rockchip-linux/rkdeveloptool/pull/85.patch

git am *.patch
autoreconf -i
./configure
make -j $(nproc)
sudo cp rkdeveloptool /usr/local/sbin/
```

### Установка на Ubuntu 22.10 и новее
Для более новых версий Ubuntu можно установить **rkdeveloptool** через пакетный менеджер:
```sh
sudo apt-get install rkdeveloptool
```

## Вход в режим Maskrom

1. Отключите карту microSD и питание устройства.
2. Установите eMMC.
3. Замкните отверстие Maskrom с помощью провода DuPont.
4. Подключите устройство к ПК через кабель USB-A — USB-A или USB-С — USB-A если позволяет материнская плата.
5. Подключите питание.
6. Если устройство обнаружено на ПК, оно вошло в режим Maskrom.


> **Примечание 1:** OTG-разъем на Radxa ROCK 5A — это порт USB 3.0 в верхней части платы. Он используется для подключения к ПК.

> **Примечание 2:** Проверить подключения можно командой
>
>  ~$ lsusb
>

## Использование **rkdeveloptool**

### Запись образа на устройство

> **Важно:** **rkdeveloptool** не распаковывает файлы автоматически. Сначала необходимо распаковать образ вручную.

1. Распакуйте образ (пример для **rock-5c_debian_bullseye-test_kde_2.img.xz**):
   ```sh
   xz -d rock-5c_bookworm_kde_b1.output.img.xz
   ```

2. Запишите загрузчик:
   ```sh
   sudo rkdeveloptool db rk3588_spl_loader_v1.08.111.bin
   ```

3. Запишите системный образ:
   ```sh
   sudo rkdeveloptool wl 0 rock-5c_bookworm_kde_b1.output.img
   ```

4. Перезагрузите устройство:
   ```sh
   sudo rkdeveloptool rd
   ```

> **Примечание:** **rkdeveloptool** не поддерживает выбор Maskrom-устройства или носителя для записи. Подключайте только одно устройство и один носитель во избежание проблем. Если требуется прошить несколько устройств, используйте **upgrade_tool**.

README: Снятие образа с eMMC и прошивка нового устройства
---------------------------------------------------------

Этот документ содержит пошаговые инструкции по созданию оптимизированного образа eMMC и его записи на другое устройство. Рекомендуется выполнять действия с liveCD или при отсутствии активных монтированных разделов на исходном устройстве.

* * *

1\. Определение разделов исходного устройства
---------------------------------------------

Для начала необходимо определить, как разделены данные на исходном устройстве. Предположим, что наше eMMC определяется как `/dev/sda`.

### Команда для просмотра разделов:

```bash
lsblk -e 7
```

_Пояснение:_  

* Опция `-e 7` исключает loop-устройства (например, snap-образы) из вывода, оставляя только физические устройства.

Пример вывода:

```plaintext
sda&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; 8:0&nbsp;&nbsp;&nbsp; 1&nbsp; 58,2G&nbsp; 0 disk
├─sda1&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; 8:1&nbsp;&nbsp;&nbsp; 1&nbsp;&nbsp;&nbsp; 16M&nbsp; 0 part&nbsp;&nbsp;&nbsp; (FAT16, конфигурационный раздел)
├─sda2&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; 8:2&nbsp;&nbsp;&nbsp; 1&nbsp;&nbsp; 300M&nbsp; 0 part&nbsp;&nbsp;&nbsp; (FAT16, EFI/загрузочный раздел)
└─sda3&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; 8:3&nbsp;&nbsp;&nbsp; 1&nbsp; 57,9G&nbsp; 0 part&nbsp;&nbsp;&nbsp; (ext4, rootfs)
```

Также можно получить дополнительную информацию о файловых системах:

```bash
lsblk -f
```

* * *

2\. Обнуление свободного места (для оптимизации образа)
-------------------------------------------------------

Перед созданием образа важно обнулить неиспользуемые данные, чтобы потом dd создавал разрежённый файл, не записывая «пустые» сектора. Для разделов с ext2/3/4 можно воспользоваться утилитой `zerofree`, для других файловых систем (например, FAT16) – методом заполнения временным файлом.

### 2.1. Обнуление свободного места для ext4 (sda3)

1. Если раздел sda3 не смонтирован, можно сразу запустить: 
   `bash &nbsp;&nbsp; sudo zerofree /dev/sda3 &nbsp;&nbsp;` 
   Если раздел не смонтирован или вы хотите выполнить проверку, можно смонтировать его в режиме «только для чтения»: 
   `bash &nbsp;&nbsp; sudo mount -o ro /dev/sda3 /mnt &nbsp;&nbsp; sudo zerofree /dev/sda3 &nbsp;&nbsp; sudo umount /mnt &nbsp;&nbsp;` 
   _Важно:_ Убедитесь, что на момент запуска утилита не изменяет данные.

### 2.2.\* Обнуление свободного места для FAT16-разделов (sda1 и sda2)

* Чаще всего эти разделы занимают мало места и занулять эти разделы необязательно

Утилита zerofree не работает с файловой системой FAT16, поэтому требуется обнуление вручную:

**Для sda1:**

1. Смонтируйте раздел: 
   `bash &nbsp;&nbsp; sudo mount /dev/sda1 /mnt &nbsp;&nbsp;`
2. Заполните свободное место нулями: 
   `bash &nbsp;&nbsp; sudo dd if=/dev/zero of=/mnt/zero.fill bs=1M status=progress &nbsp;&nbsp;` 
   Команда продолжится до заполнения раздела – после появления ошибки (недостаточно места) процесс завершите (обычно он автоматически остановится или вы можете его прервать с помощью `Ctrl+C`).
3. Удалите файл и отмонтируйте раздел: 
   `bash &nbsp;&nbsp; sudo rm /mnt/zero.fill &nbsp;&nbsp; sudo umount /mnt &nbsp;&nbsp;`

**Для sda2:**   
Повторите те же действия:

```bash
sudo mount /dev/sda2 /mnt
sudo dd if=/dev/zero of=/mnt/zero.fill bs=1M status=progress
sudo rm /mnt/zero.fill
sudo umount /mnt
```

* * *

3\. Снятие образа с eMMC
------------------------

После того, как свободное пространство на всех разделах было обнулено (и zerofree выполнена там, где это возможно), можно создавать образ устройства.

### 3.1. Создание образа с помощью dd

Используем команду `dd` с опцией `conv=sparse` для создания разрежённого файла:

```bash
sudo dd if=/dev/sda of=emmc_image.img bs=4M conv=sparse status=progress
```

Пояснения:

* **if=/dev/sda** — исходное устройство (весь eMMC).
* **of=emmc\_image.img** — имя файла образа.
* **bs=4M** — размер блока (можно настроить для оптимальной скорости).
* **conv=sparse** — создаёт разрежнённый файл, в котором нулевые блоки не занимают физическое место.
* **status=progress** — отображает ход операции.

_Замечание:_ Логический размер образа может оставаться равным полной емкости eMMC (например, 58 ГиБ или около 62 ГБ в десятичной системе), однако фактический занимаемый объем можно проверить командой:

```bash
du -h emmc_image.img
```

### 3.2. (Опционально) Сжатие образа

Для уменьшения объема файла:

```bash
gzip -c emmc_image.img > emmc_image.img.gz
```

Для восстановления:  

```bash
gunzip -c emmc_image.img.gz > emmc_image.img
```

* * *

4\. Прошивка (установка) образа на новое устройство
---------------------------------------------------

Теперь, когда образ создан, его можно записать на другое устройство (например, eMMC для нового устройства), подключенное через ридер.

### 4.1. Подготовка целевого устройства

Определите имя устройства целевой eMMC: 
   `bash &nbsp;&nbsp; lsblk -e 7 &nbsp;&nbsp;`

Убедитесь, что на новом устройстве не смонтированы никакие разделы: 
   `bash &nbsp;&nbsp; sudo umount /dev/sdX* &nbsp;&nbsp;` 
   где `/dev/sdX` — новое устройство (например, `/dev/sdb`, `/dev/mmcblk0` и т.п.).

Если требуется, сохраните или восстановите таблицу разделов. Если вы клонировали весь диск, таблица разделов уже скопирована.

### 4.2. Запись образа на целевое устройство

Используйте команду dd для записи:

```bash
sudo dd if=emmc_image.img of=/dev/sdX bs=4M conv=sparse status=progress
```

_Внимание:_ Точно укажите целевое устройство, чтобы случайно не перезаписать важные данные!

После завершения операции выполните:

```bash
sync
```

Эта команда убедится, что все данные записаны на устройство.

* * *

5\. Проверка и завершение
-------------------------

1. Извлеките устройство и установите его в новое оборудование.
2. При загрузке убедитесь, что система запускается корректно.

* * *

Дополнительные рекомендации
---------------------------

**Работа с разрежёнными файлами:**   
  Учтите, что разрежнённый файл будет иметь логический размер равный размеру исходного устройства, но фактически занимать меньше места. При переносе и записи этот файл будет восстановлен в полном объёме. 
 

**Проверка свободного места:**   
  Если образ получается слишком большим, проверьте, что действительно обнулено свободное пространство во всех разделах (особенно для FAT16, где zerofree не работает).

**Подготовка системы:**   
  Если возникают сомнения относительно работы утилит, можно провести операции с live-системы, чтобы минимизировать активность на клонируемом устройстве.
